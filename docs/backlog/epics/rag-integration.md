## 목표

Vector DB 기반 RAG 파이프라인 구축으로 포트폴리오 프로젝트 정보를 효과적으로 검색하고 응답 생성

---

## 배경

현재는 `projects.json`에서 직접 컨텍스트를 로드하지만,
프로젝트가 많아지거나 상세 정보가 늘어나면 한계가 있음.

RAG를 통해:
- 관련 정보만 선별적으로 검색
- 토큰 사용량 최적화
- 더 정확한 응답 생성
